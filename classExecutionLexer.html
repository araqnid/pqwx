Detailed Description

Lex through SQL text, breaking it into SQL commands and backslash commands.

This should basically do what psql's lexer does, although I haven't simply coped that wholesale. This is based on UTF-8 rather than wxChar, which matches how the data is stored in Scintilla, hence the use of std::string.

A lexer is created on a character buffer, and then Pull() called repeatedly, until it returns a token with type ExecutionLexer::Token::END.
